Token comes from the existing login session (localStorage key: kai_session_token).

If GasDataContext and gasApi.ts already exist and are wired:
  → Keep them, but update the BASE_URL to the URL above.
  → Update the candidates fetch to pass: page, limit, and any active filters as query params.

If they do not exist, create them as follows:

gasApi.ts:
  const BASE_URL = "https://script.google.com/macros/s/AKfycbxfNPL371bf8UF84bMz3E2i8drw4opVpWJMb24w2pW_p_og08_MwlJ5PyRqtaZPHv02Ng/exec";

  export async function fetchCandidates(token: string, params?: Record<string,string>) {
    const q = new URLSearchParams({ action: 'candidates', token, ...params }).toString();
    const res = await fetch(`${BASE_URL}?${q}`);
    return res.json();
  }

  export async function fetchCandidate(token: string, rowIndex: number) {
    const q = new URLSearchParams({ action: 'candidate', token, rowIndex: String(rowIndex) }).toString();
    const res = await fetch(`${BASE_URL}?${q}`);
    return res.json();
  }

COLUMN DEFINITIONS
==================

COL 1 — Stage
  • Badge chip with colour coding:
      "New"           → grey chip
      "Review"        → blue chip
      "Shortlisted"   → amber chip
      "Client Sent"   → purple chip
      "Selected"      → green chip
      "Deployed"      → dark green chip
      "On Hold"       → orange chip
      (any other)     → grey chip
  • Left-most column, always sticky
  • Show checkbox on hover for bulk select

COL 2 — Candidate
  • Line 1: Full name, bold, 14px
  • Line 2: Nationality · Age (e.g. "Indian · 41Y"), grey text, 12px
  • If age = 0 or missing, show nationality only
  • Sticky column (stays visible when scrolling right)
  • KI Number: NOT shown here. Do not display.

COL 3 — Passport
  • Line 1: Passport number (from candidate.passportNo field)
  • Line 2: "ECNR" or "ECR" badge (from candidate.ecrStatus field)
  • If passportNo is empty: show "—" in grey
  • If ecrStatus is empty: show nothing on line 2
  • Passport column never causes row to be hidden or greyed

COL 4 — Location
  • Single line: candidate.currentLocation
  • If empty: "—" in grey
  • No icon needed

COL 5 — Education
  • Line 1: candidate.educationLevel (e.g. "Degree", "Diploma", "ITI / Certificate")
  • Line 2: candidate.educationSubject (e.g. "B.Tech Mechanical"), grey text, 12px
  • If educationLevel empty: show educationRaw as single line (truncated at 40 chars)

COL 6 — Position Applied
  • Single line: candidate.positionApplied
  • Truncate at 30 chars with ellipsis

COL 7 — Trade / Skill
  • Single line: candidate.trade
  • Bold, slightly darker text to signal importance

COL 8 — KAI Assessment
  • Line 1: score percentage e.g. "88%" — bold, larger font (16px)
  • Line 2: tier label:
      confidenceTier === "STRONG"   → "Strong Match"  green text
      confidenceTier === "GOOD"     → "Good Match"    blue text
      confidenceTier === "POSSIBLE" → "Possible Match" amber text
      confidenceTier === "REVIEW"   → "Review"        grey text
  • Do NOT show any AI text here. Only score + tier label.

COL 9 — CV
  • Show only if candidate.cvLink is not empty
  • Three small icon buttons in a row:
      👁 Preview  → open cvLink in iframe modal or new tab
      ↗ Open     → window.open(cvLink)
      ⬇ Download → anchor download link
  • If cvLink empty: "No CV" in grey, 11px

COL 10 — Top 3 Positions
  • Show: candidate.top3Positions.first (truncated at 25 chars)
  • If top3Positions.count > 1: append " +{rest.length}" in grey
      e.g. "QA QC Lead Inspector +2"
  • On hover over this cell: show a tooltip/popover listing all positions
  • If no positions: show "—"

COL 11 — Contact
  • Show icons only when the data field is not empty:
      📞  — show if candidate.mobile is not empty
      💬  — show if candidate.mobile is not empty (WhatsApp)
      ✉   — show if candidate.email is not empty
  • Icons are action buttons (clicking opens tel: or mailto: link)
  • Do NOT show actual phone/email in the grid
  • Space icons evenly, small size (16px)

COL 12 — Recruiter Actions
  • Single "⋮" button (vertical ellipsis)
  • On click: dropdown menu appears with these items:
      "📞 Call Candidate"
      "💬 WhatsApp"
      "⭐ Shortlist"
      "📋 Request Documents"
      "📤 Submit to Client"
  • Actions are click handlers — show toast "Action: [name]" for now
  • Build with expansion in mind (future items will be added)

Call on load:
  GET ?action=candidates&token={kai_session_token}&page=1&limit=100

Response shape:
  {
    ok: true,
    records: Candidate[],   // array of candidates (see field mapping below)
    total: number,          // total matching count
    page: number,
    limit: number,
    totalPages: number
  }

Token: read from localStorage key "kai_session_token"

Use React state (useState + useEffect) to:
  • Store candidates array
  • Store isLoading boolean
  • Store error string
  • Store total count
  • Show skeleton loader rows while isLoading = true
  • Show error banner if error is not null

Field mapping from the API response:
  stage             → COL 1
