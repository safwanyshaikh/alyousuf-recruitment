CHANGE 2 — UPGRADE THE HEADER SEARCH BAR

The header search bar (the one that stays in the navbar) becomes the KAI Universal Search. Make these changes to it:

1. Change its placeholder text to:
   "Search Candidate, KI No, Passport, Mobile, Email, Requirement..."

2. Wire it so that when the user types, after a 400ms debounce and minimum 2 characters, it appends `q=<searchTerm>` to the candidates API call. When cleared, remove the q param.

3. Every time the search term changes: reset to page 1, then re-fetch.

4. Use AbortController to cancel the previous in-flight request before sending a new one.

5. Update the record count label:
   - When search is active: "Showing 1–100 of 243 results for "welder""
   - When no search: "Showing 1–100 of 5,820 candidates"

The API call format (already exists in the codebase — just add q):
```
const params = new URLSearchParams({
  action: 'candidates',
  token: sessionToken,
  page: currentPage,
  limit: 100,
  sort: 'newest',
});
if (searchQuery && searchQuery.length >= 2) params.append('q', searchQuery);
if (nationalityFilter) params.append('nationality', nationalityFilter);
if (tradeFilter) params.append('trade', tradeFilter);
// ... other filters
```

The backend handles all matching — the UI only sends q and renders what comes back. Never filter records locally.
